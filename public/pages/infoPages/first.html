- Callback -

In JavaScript, A callback is a function passed as an argument to another function. This technique allows a function to call another function and run it after another function has finished
The main purpose of using callback functions is to allow asynchronous operations to be performed, where the callback function is called once the operation has completed.

Where callbacks really shine are in asynchronous functions. This is where one function has to wait for another function.
Using callback functions allows JavaScript code to be written in a more modular and reusable way, as different functions can be written to handle different types of operations,
and then passed in as callbacks to other functions that perform those operations.

When a function is called with a callback function as an argument, it means that the function will not block the program execution, but instead it will return immediately, and the callback function will be executed once the operation is complete.

- package.JSON / Express -

"package.json" is a configuration file in Node.js projects that contains important information about the project, such as its name, version, dependencies, and scripts. It is a key file in Node.js projects that allows developers to manage dependencies and configure build processes in a standardized and automated way.
The package.json file is located in the root directory of a Node.js project and is typically created using the npm init command. It can be manually edited as well to add dependencies or scripts to the project.

The dependencies section of the package.json file lists all of the packages that the project depends on with the version numbers as well. When the project is built, these packages are automatically installed by npm.

As you can see in the picture above, the package.json file contains dependencies I am using in my project.
The dependency I will look closer at in this section is Express. Express is a web application framework for Node.js.
It is designed for building web applications and APIs.

Express allows developers to create web applications using a minimalist approach, with a focus on simplicity, flexibility, and scalability.
It also offers a wide range of plugins and extensions that can be used to add more functionality to the application, including support for authentication, database integration, and real-time communication.
